# 微信小程序提审攻略（2026 版 · 针对「成长小树苗」）

> **适用前提**：你这是**个人主体（个人号）** + **工具类目** + **无登录 / 无后端**（数据走 `wx.setStorageSync` 本地存储）的习惯打卡小程序。
> 这个前提其实很省事——**不需要任何行业资质、不需要测试账号、不需要内容安全接口**。难点集中在三件事：**基础信息合规、隐私保护指引配置、小程序备案**。
>
> **资料时效提醒**：微信官方文档更新慢、且常滞后于实际后台。下面凡标「官方」的来自微信开放文档/开放社区；标「第三方」的来自 2026 年博客/媒体，仅作趋势参考，**以你提交时后台实际要求为准**。

## 0. 先看结论：你要过哪几关、大概多久

| 关卡 | 是否必须 | 对你的难度 | 预计耗时 |
|------|---------|-----------|---------|
| ① 基础信息（名称/头像/简介）合规 | 必须 | 低（名称已合规，补头像+简介即可） | 即时 |
| ② 服务类目选「工具」并和功能一致 | 必须 | 低（个人号可开工具类，无需资质） | 即时 |
| ③ 配置《用户隐私保护指引》 | 必须 | 低（后台勾选，无需改代码） | 即时 |
| ④ 提交审核表单（功能页面+录屏+备注） | 必须 | 低（无登录→免测试账号） | 填写 10 分钟 |
| ⑤ **小程序备案（ICP）** | 必须（发布前） | 中（要身份证+人脸核身+等管局） | 平台 1–2 日 + 管局 1–20 日 |

> ⚠️ **关于 2026 备案新规（第三方说法，需核实）**：有 2026 年 3 月的媒体报道称"新增小程序**未备案无法提交审核及发布**"。但微信官方的《备案常见问答》仍表述为"**上架（发布）需先完成备案**"，未说审核本身被卡。两种口径都指向同一结论：**备案越早做越好**。建议提审通过后立刻提交备案，和审核并行，省时间。

## 1. 提审前必做的 4 件事（含「每一项的意义」）

### 1.1 基础信息：名称 / 头像 / 简介

**后台路径**：`设置` → `基本设置`

| 项 | 规则要点 | 你的「成长小树苗」怎么填 |
|----|---------|------------------------|
| **名称** | ① 不能用功能描述词（如"记账""外卖"）；② 不能"地域+广义词"；③ 不能营销词（"免费""大全"）；④ 不能含"中国/全国/官方"等（除非政府/事业单位）；⑤ 提交后改名需重新审核，30 天内最多改 2 次 | ✅ "成长小树苗"是**品牌化名称**，非功能描述、非地域词，合规。**保持不变即可**。 |
| **头像** | PNG/JPG，≥144×144；**严禁使用微信/腾讯/Apple 等官方图标或模仿其风格** | 用你的小松鼠「果果」头像即可（本地手绘水彩风，非官方图标）。确保清晰、方图。 |
| **简介** | 4–120 字；**必须具体描述核心功能**，禁止"多功能平台""一站式服务"等模糊表述 | 建议填：「帮助小学低年级孩子养成每日习惯打卡的成长小工具，支持打卡、补卡、连续天数与星星奖励。」（约 35 字，具体、和类目一致） |

> **为什么重要**：名称/头像/简介是审核第一关，模糊或违规表述会被直接驳回，且改名要重审、限频次。

### 1.2 服务类目：必须和功能严格一致

**后台路径**：`设置` → `服务类目` → `添加类目`

- **为什么重要**：审核会逐项比对"你实际做了什么"和"你选的类目"。**类目不符是直接驳回的最高频原因之一**。例如做了外卖却没选餐饮类，功能再完整也过不了。
- **个人号限制**：个人主体**只能**选工具、部分生活服务、非学历教育等；**电商、医疗、金融、社交、时政**等必须企业/组织主体，个人选不了也过不了。
- **给你的推荐**（均无需额外资质，资质栏为「/」）：

| 推荐类目 | 路径 | 理由 |
|---------|------|------|
| **主类目：备忘录（待办事项记录）** | 工具 → 备忘录 | 习惯打卡 = 每日待办记录，且"不涉及用户原创内容传播及公开访问"——你的数据全本地，完美匹配 |
| 备选：日历 | 工具 → 日历 | 统计页有日历视图，可加作第二类目覆盖功能（建议加，避免"功能超出类目"质疑） |

> ✅ 你之前选的「工具」方向正确。提交时把具体子类目定为**备忘录**，并补**日历**，两个都无需资质，个人号可开。

### 1.3 用户隐私保护指引（强制，不配接口会被禁用）

**后台路径**：`设置` → `服务内容声明` → `用户隐私保护指引` → `编辑`

- **为什么重要（两层）**：
  1. **法律合规**：《个人信息保护法》要求处理用户信息必须告知并取得同意。
  2. **技术硬约束**：微信规定——**只有在指引里声明过的隐私接口，才能调用**；未声明就调用会直接报错 `errno 112`（未声明）或被平台回收权限 `appid privacy api banned`。反过来，如果你在提审时勾了"未采集隐私"但代码其实调了隐私接口，接口会被 ban。
- **好消息**：自 2023-10-17 起，**官方隐私弹窗已自动向用户弹出**，开发者无需改代码。你只需在后台**把指引填好、声明清楚**即可。
- **你的 app 要声明什么（关键动作）**：
  1. 打开代码，搜这些隐私接口，看你是否真的调用了：
     - `wx.getUserProfile` / `<button open-type="getUserInfo">`（收集头像昵称）
     - `<input type="nickname">`（收集你选中的昵称）
     - `wx.login`（获取 openid）、`wx.chooseMedia`/`wx.chooseImage`（相册/相机）、`wx.getClipboardData`（剪贴板）、位置/手机号等
  2. 「成长小树苗」大概率只涉及**孩子昵称输入**：
     - 若用了 `<input type="nickname">` → 在指引里勾选「**收集你选中的昵称（如昵称、姓名等）**」
     - 若只是普通 `<input>` 文本输入昵称、且未调任何微信隐私接口 → 可如实勾选对应项或按最少原则声明
  3. **核心原则：指引声明 = 代码实际调用，二者必须一致**。不确定就宁可多声明，别漏。
- **隐私协议正文**：后台有模板，填好主体（你本人）+ 收集项说明即可。个人号无企业信息，照实填。

### 1.4 功能完整性自查（审核硬指标）

- 三个页面（打卡 / 统计 / 任务）**都必须是真实可用内容**，不能有"敬请期待""测试中"等占位文案、空白页。
- 所有按钮可点、无报错。
- **禁止诱导分享**：不能有"分享得红包/解锁功能"等话术（普通"分享给朋友"合规，但别诱导）。
- 你的 app 是纯本地、无 UGC（无用户发帖/评论/公开图片），所以**不需要接内容安全接口**（msgSecCheck/imgSecCheck）——这又省一事。

## 2. 提交审核表单：逐项填法（对照操作）

**后台路径**：`版本管理` → `开发版本` → 找到 v1.0.0 → `提交审核`

| 表单字段 | 必填 | 填什么 | 你的操作建议 |
|---------|------|--------|-------------|
| **功能页面** | ✅ 必填 | 选页面路径 + 上传该页截图 + 写页面功能描述 | 把 3 个页面都加上：① `pages/home/home`（打卡首页）② `pages/stats/stats`（统计）③ `pages/tasks/tasks`（任务）。每页传一张真机/模拟截图，描述写"每日习惯打卡""连续天数与星星统计""习惯任务管理" |
| **服务类目** | ✅ 必填 | 已配置的類目 | 显示「工具-备忘录 / 工具-日历」，确认无误 |
| **测试账号** | 视情况 | 功能需登录才填 | ✅ **你无需填**（无登录、纯本地）。提交时该项留空/注明"无需登录"即可，不会因此驳回 |
| **操作录屏** | 建议 | 30 秒内功能演示视频（MP4） | 建议录一段：打开→打卡今日习惯→看统计页→加一个任务。无登录更方便录，能帮审核员快速理解，降低驳回概率 |
| **备注说明** | 选填但建议 | 本版本核心功能/改动 | 模板：「v1.0.0 习惯打卡工具，面向小学低年级学生。支持每日打卡、历史补卡、连续天数与星星奖励、习惯任务管理。数据本地存储，无登录无后端。已配置隐私保护指引。」 |
| **标签** | 选填 | 搜索标签 | 可填「习惯」「打卡」「成长」「儿童」 |
| **反馈页面** | 选填 | 用户反馈入口 | 个人号可先不填 |

> 提交后进入审核队列。普通类目通常 **1–7 个工作日**。审核结果会发到管理员微信（模板消息）+ 后台「通知中心」。

## 3. 审核通过后、发布前：小程序备案（2026 硬门槛）

- **为什么必须**：微信官方错误码 `86369` = "小程序上架需先完成备案"。**不备案点不了"发布"**。
- **顺序建议**：`提交审核` → `审核通过` → `提交备案`（与审核并行最省时）→ `备案通过` → `发布`。
- **个人主体备案材料**（后台路径：`微信公众平台` → 顶部/`设置` → `小程序备案`）：
  - 身份证（系统自动识别姓名/地址/证件号，需与上传一致）
  - 通讯地址（精确到门牌号；若身份证地址与备案省份不一致，部分省份需暂住证/居住证：吉、沪、苏、浙、皖、鲁、鄂、粤、川、黔、滇）
  - 主体负责人 = 小程序管理员本人，**人脸核身**（需中国大陆实名微信号扫码）
  - 《小程序负责人授权书》《互联网信息服务核准承诺书》（后台有模板，首次签署半年有效）
  - 服务内容选**工具类（非经营/非企业）**——个人不能选经营性质
- **周期**：平台初审 1–2 工作日 → 管局审核 1–20 工作日（各省不同）。提交管局后会收到 **12381 短信，24 小时内必须核验**，否则退回重来。
- ⚠️ 备案期间小程序**已审核通过但还不能向公众发布**，耐心等管局即可。

## 4. 高频驳回原因 & 本项目的针对性自查

| 驳回类型 | 你的风险 | 对策 |
|---------|---------|------|
| 类目不符 | 低（工具-备忘录匹配） | 确认子类目=备忘录+日历，不越界到教育/社交 |
| 功能不完整（空白页/占位） | 低 | 三页真实内容，无"测试中" |
| 测试信息缺失 | 无（无登录） | 测试账号留空并注明 |
| 名称/头像/简介不合规 | 低 | 名称不变；头像用手绘松鼠非官方图标；简介按 1.1 填具体 |
| 隐私接口未声明 | **中** | 按 1.3 搜代码，指引里如实声明昵称等，做到"声明=调用" |
| 诱导分享 | 低 | 检查文案，无"分享得奖励" |
| 未成年人内容 | **需留意** | 面向 7–8 岁儿童的 app：避免收集真实姓名/人脸/证件号，只用本地昵称；不宣称医疗/教育资质。目前定位"工具"最稳 |

- [ ] **服务类目**：工具→备忘录（主）+ 工具→日历（备），确认无资质要求
- [ ] **隐私指引**：后台勾选「收集你选中的昵称」（若代码用了 nickname 输入）；其余按代码实际调用补；确保"声明=调用"

- [ ] **上传/提交**：开发版 v1.0.0 → 提交审核；功能页面加 3 页截图+描述；传 30s 录屏；备注按模板填

- [ ] **备案**：审核一通过（或并行）立刻提交备案，备齐身份证+地址+人脸核身，24h 内验 12381 短信
