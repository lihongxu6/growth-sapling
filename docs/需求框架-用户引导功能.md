## 1. 背景

- v2.0 已提交审核（2.0.1），含开屏页 + 成长页用户信息区（本地版，不接云）。
- 现状缺口：用户（儿童）首次进入后，不知道从哪开始、数据去哪了、为何要相册/摄像头授权。
- §3.15 已决策：首次轻引导 + 隐私透明提示统一放到下一版「小程序使用引导规划」，从界面设计整体做。
- 2026-07-20 用户确认：**下一版本 = 用户引导功能**，并给出具体范围（见 §2 / §4）。

---

## 2. 目标

- **G-目标1**：儿童首次进入，通过「跟着做一遍」的引导掌握核心操作（建任务 / 打卡 / 补卡 / 看日历 / 看徽章 / 改头像昵称）。
- **G-目标2**：引导中透明说明「数据仅存本机、不上传」，并在调用相册/摄像头前弹授权说明。
- **G-目标3**：引导所建的任务沉淀为**真实账户数据**（不删）——引导即首次真实使用。
- **G-目标4**：成长页提供常驻「操作手册」入口，随时可回看。
- **G-目标5**：不打断既有已验证界面（复用真实页面 + 引导覆盖层，§3.8 / §3.10）。

---

## 3. 用户与场景

- **主用户**：7–8 岁儿童（**受众确认为儿童**，文案童趣 / 图文为主；家长可协助）。
- **场景 A（冷启动 · 首次引导）**：清空数据 → 引导儿童走完整核心流程（建任务→打卡→补卡→看日历→看徽章→改头像昵称）。
- **场景 B（授权疑虑）**：改头像时弹相册/摄像头授权，引导已提前说明原因。
- **场景 C（随时回看）**：使用中忘记某功能，从成长页操作手册图标查看图文说明。

## 6. 关键规则（建议，待确认）

- **R1 清空保护**：仅「首次引导启动且用户确认」时清空数据；帮助页 / 重新看引导**不触发清空**；中途退出引导**保留已操作进度**（不回滚）。
- **R2 引导即真实使用**：引导所建任务 = 真实账户任务，走完即首个真实数据，不再删除。
- **R3 受众童趣**：文案用儿童语言 + 图文 / emoji，避免长句与抽象术语。

---

## 7. 待确认（≤3 设计分叉）

1. **帮助页形态**：操作手册是纯静态图文步骤说明，还是也带「重新走一遍交互引导」按钮？
2. **首次引导可跳过**：儿童 / 家长能否「跳过」直接进小程序，还是必须走完才能用？
3. **引导步骤强控**：必须按顺序完成才进下一步（强引导），还是仅高亮提示、可自由操作（弱引导）？

---

## 8. 验收标准（草案）

- 首次进入触发引导，清空数据后儿童能跟着走完核心流程；
- 引导所建任务存在于账户、不被删；
- 改头像时授权弹窗文案与引导一致；
- 成长页有操作手册图标，点击可看图文说明；
- 不新增独立「教程中心」页（用单页操作手册实现，§3.8）。
