## 1. 设计原则

| 原则 | 说明 | 来源 |
|------|------|------|
| 儿童友好（7-8 岁） | 具象图形、大色块、大点击区；识字量 1000-1500 字，文案全面向孩子 | PRD 目标用户 / design-review 一 |
| 成长伙伴而非监督 | 温暖、鼓励、不惩罚；不用"管理员""家长"等词；IP 小松鼠"果果" | design-review 一 |
| 克制不花哨 | 动效轻量（≤1500ms）、对比清晰、避免分散注意力 | design-review 五（动画 checklist） |
| 自我管理基调 | 数据可视化（火焰/星星/徽章）让孩子看见自己的成长，而非被催促 | PRD 核心理念 |

## 3. 组件库规格

每个组件给出视觉定义与状态变体。状态变体命名与 design-review「状态清单」对齐。

### 3.1 状态栏（Status Bar）
- 高度 44px 的**顶部安全区**；**不显示品牌名、假时间、信号/电量图标**，保持极简。
- 常驻顶部，不随内容滚动；为 iOS 状态栏/安全区预留空间。

### 3.1.1 开屏页（Splash）
- 首次启动展示：全屏居中 `🌱` + 产品名 `成长小树苗` + slogan（如"陪你慢慢长大"）。
- 停留约 1.5s 后淡出（600ms）进入首页；点击可提前跳过。
- 用途：强化品牌第一印象，并为云开发初始化/数据加载争取时间。
- 日常流程（含各 Tab）不再重复展示品牌名，避免与页面标题视觉冲突。

### 3.2 打卡页顶部信息栏（Home Header）
- 三栏左→右按重要度排列：`🔥 连续天数`（左，最重要）/ `⭐ 星星数`（中，居中）/ `📅 日期`（右，点按展开日历）。
- 火焰：当日无完成 = `🪵` 黑白未点亮；当日有完成 = `🔥` 彩色 + 发光动画。
- 吸顶常驻（`position: sticky; top:0`），背景 `--bg`。

### 3.3 任务卡片（Task Card）
| 状态 | 视觉 |
|------|------|
| 待打卡 | 白底 `--card-bg` + 1px `--border` 边框 + 右侧空心圆 ○ |
| 已完成 | `--green-light` 底 + 右侧 `--green` 对勾 ✅ |
| 补卡 | 右上角"补"标签（`--orange-tag` 底、`--yellow` 字）+ 卡片淡橙底 |
- 结构：图标（左，28px）/ 名称（中，`--fs-card-title`）/ 状态图标（右）。
- 最小点击区 44×44px；卡片整体可点（打卡/撤销打卡）。
- 状态切换：背景 300ms ease-out 过渡（白↔浅绿）。

### 3.4 进度条（Progress）
- 文字 `已完成 X/N` + 条形进度（圆角 `--radius-pill`，填充 `--green`）。
- 星星总数仅在顶部信息栏展示，进度条下方**不重复**显示星星。

### 3.5 空状态（Empty State）
- 小松鼠"果果" + 引导文案 + 主按钮。
  - 打卡页：`来添加你的第一个任务吧！` + `去添加任务` 按钮（跳任务页）。
  - 任务页：`点击下方按钮，添加你的第一个任务吧！`
  - 统计页：`坚持打卡，这里会显示你的成长记录哦～`
- 果果表情：招手（空状态）、鼓励（断签）、祝贺（徽章）、日常（加载）。**当前以 emoji 🐿️ 占位，正式插画待外部插画师产出（design-review 待确认问题 #1）。**

### 3.6 统计页模块
- **连续天数**：`🔥 连续 X 天` 大数字（`--fs-display`，`--yellow`），下方 `最长 X 天`（最显眼元素）。
- **本周完成率**：环形图 + 百分比（中心留白显示数字）。
- **月历热力图**：7 列网格，每格颜色 绿/黄/灰/白；支持左右滑切换月份；点击某天（未来不可点）跳转打卡页对应日期（design-review 手势表 297）。
- **徽章墙**：2 行 × 4 列（7 枚 + 1 预留"敬请期待"）。每枚：图标 + 名称 + 进度文字。未获得=灰+锁；已获得=彩+发光边框。

### 3.7 任务管理卡片（Task Mgmt Card）
- 图标 + 名称 + `💭 任务意义`（选填，`--blue` 斜体）+ 周期描述（`--text-secondary`）+ 编辑 ✏️ + 删除 🗑️。
- 点击卡片 = 编辑；点击删除 = 确认弹窗（防误触）。

### 3.8 添加/编辑任务表单（Modal）
- 标题 `添加任务` + 关闭 ✕。
- 字段：任务名称（输入）、`为什么要做这个任务？（选填）` 输入（purpose，最长 50 字，PRD 9.1）、图标 12 宫格单选、重复周期单选（每天/工作日/周末/自定义）、自定义星期勾选、保存按钮。
- 输入框：白底 + `--border` 边框 + `--radius-md`；聚焦态 `--green` 边框。

### 3.9 日历弹层（Calendar Overlay）
- 月视图，标题 `◀ 2025年3月 ▶`。
- 按时 = 实心 ●，补卡 = 空心 ○，今天 = 高亮边框，未来 = 不可点（置灰）。

### 3.10 确认弹窗（Confirm Modal）
- 撤销打卡：`确定撤销「XXX」的打卡吗？`；删除任务：`确定删除「XXX」吗？之前的打卡记录会留着哦～`。
- 主按钮（危险操作用 `--danger`）+ 取消（次级）。

### 3.11 Toast
- 底部上浮提示，圆角胶囊，`--fs-caption`，2s 自动消失。

### 3.12 底部导航（Tab Bar）
- 3 项：🏠 打卡 / 📋 任务 / 👤 我的成长；当前项 `--green`。「统计/成长」页在 v2.0 承载用户信息与退出登录，移至最右侧符合"个人信息在右下"的通用习惯；标签由「成长」更名「我的成长」（v2.0 决策，2026-07-18，见 MEMORY §3.9）。
- **常驻贴底**，不被 iPhone 安全区/home indicator 遮挡（`padding-bottom: env(safe-area-inset-bottom)`）。
- 图标 ≥ 22px，标签 ≥ 11px，整体点击区 ≥ 44px。

### 3.13 庆祝 / 徽章动画（Overlay）
- 全部完成：`🌟 太棒了！🌟` 大字 + 左右摇摆 + 果果，1500ms。
- 徽章获得：灰→彩 + 全屏光效 + 果果弹入，1500ms，不打断操作。

## 5. 交互动效规范

> 参数直接引自 design-review 四「动画规格」。

| 动画 | 时长 | 缓动 | 说明 |
|------|------|------|------|
| 打卡星星迸发 | 600ms | ease-out | 3-5 颗星向四周扩散，透明度 1→0 |
| 星星数 +1 | 200ms | ease-out | 顶部星星数字弹跳 +1 |
| 全部完成庆祝 | 1500ms | ease-in-out | 文字缩放 + 左右摇摆 |
| 徽章点亮 | 1500ms | ease-out | 灰→彩 + 光效扩散 + 果果弹入 |
| 徽章回收 | 600ms | ease-in-out | 彩→灰 + 缩小至 90% |
| 卡片状态切换 | 300ms | ease-out | 白↔浅绿背景过渡 |
| 日历月切换 | 300ms | ease-in-out | 左右滑动 |

**约束**：所有动画 ≤1500ms，不阻塞操作；低端机可降级（design-review 评审议题 #3）。

## 6. 响应式 / 自适应

- 设计基准 375×812（iPhone）；最小点击区 44×44px（≈88rpx）。
- 页面容器：移动端 `width:100%; height:100dvh;`，状态栏/底部导航 `flex-shrink:0` 常驻，中间内容区 `overflow-y:auto`。

- 小程序注意：`100dvh` 在小程序需改为 `100vh` + `env(safe-area-inset-bottom)` 适配（技术方案阶段专项处理）。

## 9. 与技术方案衔接（下轮输入）

| # | 逻辑 | 规则 | 触发条件 |
|---|------|------|----------|
| L1 | 今天不显示「补」标签 | 仅当 `viewDate < today`（补卡模式）才显示补卡标签 | 渲染首页任务卡片 |
| L1.1 | 补卡模式打勾自动标 backfill | `viewDate < today` 下点未完成 → 标 done + backfill=true；该卡显示「补」标签 + 淡橙底 `--orange-tag` | `onTaskClick` 写盘逻辑 |
| L1.2 | 按期打卡不标 backfill | 仅 `viewDate < today` 时完成的才设 `backfill=true`；`viewDate === today` 的打卡 `backfill=false`，不显示「补」标签 | `onTaskClick` 写盘逻辑 |
| L1.3 | 撤销补卡清除 backfill | 撤销打卡时同步 `t.backfill=false`，补卡标签消失 | `confirmOk.onclick` |
| L1.4 | 今日永不显示补卡 | `renderHome` 入口判断：`viewDate === today` 时强制 `homeTasks.forEach(t=>t.backfill=false)`，杜绝残留 | `renderHome` |
| L1.5 | 今日打卡数据与统计页数据源打通 | 今日 SSOT = `homeTasks`；`getCheckin(today)` 实时从 `homeTasks` 计算 done/total；统计页 11号 必须与打卡页一致 | `getCheckin` |
| L1.6 | 星星数随打卡操作联动 | `STARS` 不再是常量，改为 `calcStars()`：基础 7 颗 + 全部历史日已打卡数（含今日）累加；勾选/撤销时 `homeTasks.done` 变化 → 顶部 `⭐` 立即变化 | `calcStars` |
| L2 | 补卡模式 banner | `viewDate < today` 时显示「📅 补卡模式 · 日期」 | 渲染首页 |
| L3 | 撤销打卡二次确认 | 已完成状态点任务 → 弹确认弹窗，文案区分「打卡 / 补卡」 | `onTaskClick(id)` 且 `task.done === true` |
| L4 | 未完成任务直接打卡 | 未完成点任务 → 立即标完成 + 触发完成庆祝 | `onTaskClick(id)` 且 `task.done === false` |
| L5 | 未来日期不可点 | `viewDate > today` 任务列表不可交互 | `onTaskClick` 入口检查 |
| L6 | 日历点击切换日期 | 首页日历、统计页日历的过去日期格可点 → 设置 `viewDate` + 跳回首页 | `pickDate(d)` |
| L7 | 日历未来日期不可点 | 未来日期在日历中置灰、无 `onclick` | `renderCal` / `renderStats` |
| L8 | 统计页日历默认当前月 | 渲染统计页时按 `new Date()` 计算月 | `renderStats` |
| L8.1 | 统计页日历显示日期数字 | 每格中央显示日号（1-31），保留热力色块；未来日期透明背景 + 浅灰字 | `renderStats` |
