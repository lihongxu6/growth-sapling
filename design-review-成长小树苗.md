## 一、产品核心信息（设计师需理解）

### 产品定位
面向 7-8 岁小朋友的习惯养成打卡工具。核心理念：**不是监督工具，是成长伙伴。**

### 目标用户
- 主要使用者：小学二年级小朋友（识字量约 1000-1500 字，能独立操作）
- 认知特点：具象思维，对图形/色彩/动画敏感，注意力约 15-20 分钟

### 设计基调
- 温暖、鼓励、不惩罚
- 所有文案面向孩子（不用"管理员""家长"等词）
- IP 形象：小松鼠"果果"——孩子的成长伙伴

## 三、状态清单

### 打卡页状态

| 状态 | 触发条件 | 视觉表现 |
|------|----------|----------|
| 正常态 | 有任务、有未完成项 | 任务列表 + 进度条 |
| 全部完成 | 所有今日任务已打卡 | 触发庆祝动画，所有卡片绿色 |
| 空状态（无任务） | 从未创建过任务 | 小松鼠 + "来添加你的第一个任务吧！" + 按钮 |
| 休息日 | 有任务但今天都不出现 | "今天没有安排任务，好好休息吧～" |
| 补卡模式 | 选择了过去日期 | 顶部显示 "📅 补卡模式 · X月X日" |
| 加载中 | 数据请求中 | 骨架屏（3-4 个卡片占位 + 闪烁） |
| 网络错误 | 请求失败 | 错误提示 + 重试按钮 |

### 统计页状态

| 状态 | 触发条件 | 视觉表现 |
|------|----------|----------|
| 正常态 | 有打卡记录 | 连续天数 + 完成率 + 日历 + 徽章墙 |
| 空状态 | 没有任何打卡记录 | 小松鼠 + "坚持打卡，这里会显示你的成长记录哦～" |
| 徽章全灰 | 未获得任何徽章 | 7 枚徽章全灰 + 锁定 + 进度文字 |
| 加载中 | 数据请求中 | 骨架屏 |
| 网络错误 | 请求失败 | 错误提示 + 重试 |

### 任务页状态

| 状态 | 触发条件 | 视觉表现 |
|------|----------|----------|
| 正常态 | 有任务 | 任务列表 + 添加按钮 |
| 空状态 | 无任务 | 小松鼠招手 + 引导文案 |
| 达到上限 | 已有 6 个任务 | 添加按钮置灰 + 提示文字 |

| 状态 | 视觉 |

| 已持有 | 彩色 + 发光边框 |
| 被回收 | 彩色 → 灰色柔和过渡 + 轻微缩小 |

## 四、交互边界

### 动画规格

| 动画 | 时长 | 缓动 | 说明 |
|------|------|------|------|
| 打卡星星迸发 | 600ms | ease-out | 3-5 颗星从卡片中心向四周扩散，透明度 1→0 |
| 星星数 +1 | 200ms | ease-out | 顶部星星数字弹跳 +1 |
| 全部完成庆祝 | 1500ms | ease-in-out | 文字缩放 + 左右摇摆 |
| 徽章点亮 | 1500ms | ease-out | 灰色→彩色渐变 + 光效扩散 + 果果弹入 |
| 徽章回收 | 600ms | ease-in-out | 彩色→灰色 + 缩小至 90% |
| 卡片状态切换 | 300ms | ease-out | 白色↔浅绿色背景过渡 |
| 日历月切换 | 300ms | ease-in-out | 左右滑动 |

### 手势交互

| 页面 | 手势 | 行为 |
|------|------|------|
| 打卡页 | 点击任务卡片 | 打卡/撤销打卡 |
| 打卡页 | 点击日期 | 展开日历月视图 |
| 打卡页 | 下拉 | 刷新数据 |
| 统计页 | 左右滑动 | 切换日历月份 |
| 统计页 | 点击日历格 | 跳转到打卡页对应日期，定位该天任务列表（未来日期不可点击） |
| 统计页 | 点击徽章 | 展示徽章详情（获得条件、获得日期） |
| 任务页 | 点击任务卡片 | 进入编辑 |
| 任务页 | 点击删除图标 | 弹出确认弹窗 |
| 任务页 | 点击添加按钮 | 打开添加任务表单 |

### 补卡交互细节

- 日期选择器中：按时打卡日期 = 实心圆点 ●，补卡日期 = 空心圆环 ○
- 打卡页补卡模式：顶部提示条 "📅 补卡模式 · 3月14日"，背景微黄
- 统计页日历：颜色（绿/黄/灰/白）不区分补卡与按时——两者同等计入
- 卡片右上角"补"标签仅在补卡产生的打卡记录上出现

### 徽章系统交互

- 徽章墙布局：2 行，每行 4 列（7 枚 + 1 个"敬请期待"预留位）
- 徽章获得时：任何页面都可以弹出（不限于统计页），不打断当前操作
- 徽章回收：静默发生，无弹窗，仅在统计页可见变化
- 进度文字：积分类显示"还差 X 颗星"，行为类显示"还需连续 X 天"

### 设计师需要产出的交付物

| # | 交付物 | 说明 |

| 4 | 图标库 | 12-16 个任务图标（手绘风格） |

| 6 | 动效说明 | 动画参数文档（时长、缓动曲线、触发条件） |

### 设计验收 Checklist

**通用：**
- [ ] 所有文案面向 7-8 岁儿童，无"管理员""家长"等措辞
- [ ] 字体大小适合儿童阅读（建议正文 ≥ 28rpx）
- [ ] 点击区域足够大，适合小朋友手指（建议 ≥ 44×44pt）
- [ ] 颜色对比度满足无障碍标准
- [ ] 底部导航在不同尺寸手机（含 iPhone 安全区/底部 home indicator）下始终贴底显示，不被遮挡或截断

**打卡页：**
- [ ] 顶部信息栏按重要程度排列：火焰+连续天数（左）、⭐星星积分（中）、日期（右）
- [ ] 火焰图标在无完成任务时显示🪵（黑白未点亮），有完成任务时显示🔥（彩色+发光动画）
- [ ] 累计星星数在进度条下方可见
- [ ] 已完成/未完成任务视觉区分明显
- [ ] 补卡模式有明显的视觉提示（不会误以为在操作今天）
- [ ] 空状态引导清晰，可一键跳转创建任务

**统计页：**
- [ ] 连续天数是页面最显眼的元素
- [ ] 日历颜色逻辑正确（绿/黄/灰/白四种）
- [ ] 补卡日期的视觉标记与按时打卡可区分
- [ ] 徽章灰→彩的差异足够明显，灰度徽章不"丑"
- [ ] 徽章进度文字清晰可读

**任务页：**
- [ ] 添加按钮突出（主色、大尺寸、底部固定）
- [ ] 编辑入口直观（点击卡片即编辑）
- [ ] 任务意义字段选填，蓝色斜体显示在任务名称下方
- [ ] 删除有二次确认，不易误操作
- [ ] 周期选择器对儿童友好（大按钮、清晰选项）

**动画：**
- [ ] 打卡星星动画不过度（不干扰连续打卡）
- [ ] 徽章点亮动画有"惊喜感"
- [ ] 徽章回收动画温和（不引起负面情绪）
- [ ] 动画总时长不阻塞操作（建议所有动画 ≤ 1500ms）

## 六、研发关注点

### 技术依赖

| 依赖 | 说明 | 风险 |
|------|------|------|
| 微信云开发 | 云函数 + 云数据库 | 低——成熟方案 |
| 微信小程序 Canvas/动画 | 星星动画、徽章点亮动画 | 中——动画性能需关注低端机 |
| 日历组件 | 需自研或使用第三方 | 中——需要支持滑动、颜色标记、空心/实心区分 |
| 骨架屏 | 加载占位 | 低——标准方案 |

### 接口设计（云函数）

| 云函数 | 功能 | 说明 |
|--------|------|------|
| getDailyTasks | 获取今日任务列表 | 按周期过滤 + 合并当日打卡状态 |
| checkin | 打卡 | 写入 checkins + 判断 is_backfill |
| undoCheckin | 撤销打卡 | 删除记录 + 重算积分/徽章 |
| getStats | 获取统计数据 | 连续天数、完成率、日历数据 |
| getBadges | 获取徽章状态 | 全部徽章 + 获得状态 + 进度 |
| manageTask | 任务增删改 | 统一入口 |
| getHistoryDate | 获取指定日期任务 | 补卡用 |

### 数据一致性风险

| 风险点 | 说明 | 建议 |
|--------|------|------|
| 打卡并发 | 快速连续点击同一任务 | 前端按钮即时禁用 + 唯一索引兜底 |
| 撤销重算 | 撤销一条旧记录需重算全部统计 | 云函数中事务处理，或设计为幂等重算 |
| 徽章状态 | 撤销可能导致多枚徽章同时回收 | 一次性全量重判，避免逐枚通知 |
| 周期变更 | 修改任务周期后的统计口径 | 历史数据不变，仅影响未来 |
